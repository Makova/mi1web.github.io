Nombre: prueba (o el que se quiera).

Hoy en día cualquier desarrollo informático, incluyendo páginas web, debe de estar en un sistema de control de versiones tal como GitHub. Eso permite hacer cambios, saber quién los ha hecho, revertir a versiones anteriores, y por supuesto desplegar la web usando buenas prácticas en vez de ñapas como copiar .zip. No es la única forma de publicar páginas web. Se puede usar Google Sites (aunque no se tiene mucho control), Dropbox o Google Drive. GitHub te premite publicar páginas sin publicidad y con todo el código controlado por ti.

index.html

<!doctype html>
<html>
  <head>
    <title>Mi primera página</title>
  </head>
  <body><h1>Título que aparecerá en la página</h1>
    <p>Texto de la página</p>
  </body>
</html>

Configuremos la página

HTML == lenguage de descripción estructural

Lo que implica que no se describe la apariencia de la página, sino su contenido y la estructura del mismo: esto es la cabecera, esto es una parte, esta parte está aparte de esa parte... Es importante tener esto en cuenta, porque tiene todo tipo de repercusiones a la hora de crear una página fácilmente renderizable por todo tipo de navegador y analizable por los buscadores para encontrar su contenido.
